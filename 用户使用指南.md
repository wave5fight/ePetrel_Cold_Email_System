# ePetrel Cold Email System 用户使用指南

本文档面向使用新版 Web 控制台的用户，说明如何启动系统，以及如何使用每一个页面。

新版控制台基于 FastAPI + Jinja2 + Tailwind 构建，默认语言为英文，右上角可切换 English / 中文。

## 1. 启动系统

### 1.1 安装依赖

在项目根目录执行：

```bash
pip install -r requirements.txt
```

### 1.2 启动新版控制台

```bash
uvicorn web_app:app --host 127.0.0.1 --port 8000
```

浏览器打开：

```text
http://127.0.0.1:8000
```

### 1.3 推荐环境变量

生产或正式使用时，建议配置：

```bash
EPETREL_SESSION_SECRET="请替换为足够长的随机字符串"
EPETREL_BFF_BASE_URL="https://bff.epetrel.com"
EPETREL_SITE_URL="https://epetrel.com"
```

如需使用 AI 功能，也可以在 LLM Settings 页面配置 API key，不一定要写入 `.env`。

## 2. 页面总览

左侧导航包含 5 个页面：

- Dispatch Control：冷邮件发送控制台
- Security Monitor：发件安全监控
- Audit Logs：历史发送审计
- Shared Inbox：统一共享收件箱
- LLM Settings：大模型配置

右上角包含：

- 系统状态
- English / 中文语言切换
- Deploy 按钮，目前作为界面操作入口预留

## 3. Dispatch Control 冷邮件发送控制台

该页面用于配置发件箱、执行 Gmail 落箱测试、上传客户名单并启动冷邮件发送队列。

### 3.1 Mailforge Sender Pool 发件箱池

用途：添加和查看可用于发送邮件的发件箱。

字段说明：

- Email：发件邮箱地址
- Password / App Password：邮箱 SMTP 密码或应用专用密码
- Daily Limit：该发件箱每天最多发送数量
- From Name：收件人看到的发件人名称

操作步骤：

1. 在发件箱表单中填写邮箱、密码、每日上限和发件人名。
2. 点击 Save Sender。
3. 保存成功后，发件箱会显示在列表中。

注意事项：

- 邮箱必须是有效邮箱格式。
- 密码会用于 SMTP/IMAP 连接，请使用服务商推荐的 App Password。
- 发件箱状态为 active 且未达到每日上限时，才会进入发送池。

### 3.2 Managed Gmail Placement Test 托管 Gmail 落箱测试

用途：先向 ePetrel 后端提供的 Gmail 测试邮箱发送一封邮件，检查进入主邮箱还是 Spam。

操作步骤：

1. 点击 Start ePetrel Authorization。
2. 如果页面出现 Open ePetrel Signup / Login，点击后到 ePetrel 注册或登录。
3. 回到本系统，点击 Check Authorization。
4. 授权成功后，系统会显示授权账号和测试 Gmail 地址。
5. 选择 Sender to Test。
6. 填写 Test Subject Prefix。
7. 勾选 Wait for result 可在发送后等待 BFF 返回结果。
8. 点击 Send Gmail Placement Test。
9. 如需重新查询结果，点击 Refresh Placement Result。

测试邮件会带：

- subject 中的 request id
- `X-ePetrel-EmailTestRequestId`
- `X-ePetrel-Test-Sender`

注意事项：

- 必须先配置至少一个 active 发件箱。
- 必须完成 ePetrel 授权。
- BFF 地址由 `EPETREL_BFF_BASE_URL` 控制。

### 3.3 Load Target Leads 上传目标客户名单

用途：上传待发送的客户名单。

支持格式：

- `.csv`
- `.xlsx`

必要字段：

- `Email`

推荐字段：

- `Name`
- `Company`
- `Company_Bio`
- `Position`

这些字段可在邮件主题和正文中作为变量使用，例如：

```text
{Name}
{Company}
{Company_Bio}
{Position}
```

### 3.4 Configure Copy Variants 配置邮件文案

用途：配置邮件主题和 HTML 正文。

默认主题示例：

```text
{Hi|Hello} {Name}, quick idea for {Company}
```

默认正文示例：

```html
<p>{AI_Icebreaker}</p>
<p>I am reaching out from ePetrel AI Studio with a concise collaboration idea for {Company}.</p>
<p>Would it make sense to send a few examples?</p>
```

支持能力：

- 字段变量替换，例如 `{Name}`、`{Company}`
- Spintax，例如 `{Hi|Hello|Hey}`
- AI 破冰句变量 `{AI_Icebreaker}`

注意事项：

- 正文建议使用 HTML。
- 系统会自动生成 plain text 版本。
- 系统会追加退订说明，降低合规风险。

### 3.5 Flow Control 控流与队列控制

字段说明：

- Min Delay：每封邮件之间的最小等待秒数
- Max Delay：每封邮件之间的最大等待秒数
- Variant Tag：本次文案版本标签，写入发送日志
- Seed interval：每发送 N 个真实联系人后插入 1 封 seed 测试邮件
- AI realtime icebreaker：是否调用 LLM 生成实时破冰句
- Mix seed test inboxes：是否混入 seed 测试邮箱

操作步骤：

1. 上传客户名单。
2. 配置主题和正文。
3. 设置发送间隔和版本标签。
4. 如需 AI 破冰句，先到 LLM Settings 配置 API key。
5. 点击 Start Dispatch Queue。

注意事项：

- 发送队列会真实投递邮件，请先用少量数据测试。
- 若发件箱达到每日上限，系统会自动跳过。
- 若目标邮箱在 suppression list 中，系统会跳过。
- 若发件箱连续失败达到阈值，系统会自动暂停该发件箱。

## 4. Security Monitor 发件安全监控

该页面用于监控发送健康度、退信、退订、seed 落箱情况。

### 4.1 Seed Test Inbox Pool

用途：配置用于检测落箱结果的 seed 测试邮箱。

字段说明：

- Seed Email：seed 邮箱地址
- IMAP Password / App Password：seed 邮箱 IMAP 密码
- Provider：服务商，例如 Gmail
- IMAP Host：IMAP 地址，例如 `imap.gmail.com`
- IMAP Port：通常为 `993`
- Inbox Folder：主收件箱文件夹，通常为 `INBOX`
- Spam/Junk Folder：垃圾箱文件夹，例如 `Spam`
- Status：`active` 或 `paused`

操作步骤：

1. 填写 seed 邮箱的 IMAP 信息。
2. 点击 Save Seed Inbox。
3. 保存成功后，该邮箱会进入 seed 池。

### 4.2 Sync Seed Placement Now

用途：立即同步 seed 邮箱，检查已发送到 seed 邮箱的邮件进入 Inbox 还是 Spam。

字段说明：

- Stats Window / Days：统计最近多少天的数据
- Emails scanned per seed folder：每个 seed 文件夹扫描多少封邮件

操作步骤：

1. 设置统计窗口和扫描数量。
2. 点击 Sync Seed Placement Now。
3. 查看顶部指标卡和事件明细。

### 4.3 指标卡说明

- Successful Sends：成功发送数量
- SMTP Failed：SMTP 失败数量
- Bounce Rate：总退信率
- Hard Bounce：硬退信率
- Unsubscribe Rate：退订率
- Seed Spam：seed 邮箱进入 Spam 的比例

### 4.4 表格说明

Sender / Domain Summary：

- 展示最近窗口内的发送记录、目标域名、状态和错误原因。

Event Details：

- 展示退信、退订、seed inbox、seed spam、seed missing 等事件。

Sender Health：

- 展示发件箱每日发送数量、失败次数、状态和最近发送时间。

## 5. Audit Logs 历史发送审计

该页面用于查看所有出站邮件记录，并审查某封邮件的原始 HTML。

### 5.1 发送日志列表

表格字段：

- ID：发送记录 ID
- Time：发送时间
- Sender：发件箱
- Receiver：收件人
- Domain：目标域名
- Subject：邮件主题
- Status：发送状态
- Message-ID：邮件 Message-ID

### 5.2 Raw Email Render Trace

用途：查看某封邮件最终渲染后的 HTML。

操作步骤：

1. 在 Email ID to inspect 输入日志 ID。
2. 点击 Fetch Raw HTML。
3. 右侧 iframe 会显示该邮件 HTML。

适用场景：

- 检查变量是否正确替换
- 检查 Spintax 是否正常解析
- 检查最终邮件排版
- 排查某封邮件失败或投诉前的内容

## 6. Shared Inbox 统一共享收件箱

该页面用于聚合所有 active 发件箱的回信，并进行意图分类。

### 6.1 Sync Inbox Now

字段说明：

- Recent emails per mailbox：每个发件箱拉取最近多少封邮件

操作步骤：

1. 设置每个邮箱要拉取的最近邮件数量。
2. 点击 Sync Inbox Now。
3. 系统会通过 IMAP 读取 active 发件箱的 INBOX。
4. 新邮件会写入统一收件箱。

### 6.2 回信列表

表格字段：

- Received：收到时间
- Sender：客户邮箱
- Receiver：我方发件箱
- Subject：邮件主题
- Sentiment：AI 分类结果

分类结果可能包括：

- `[Interested]`
- `[Refused]`
- `[Follow Up Later]`
- `Pending`

### 6.3 自动退订识别

系统会识别以下类似表达：

- unsubscribe
- remove me
- stop emailing
- do not contact
- opt out

识别到退订后，系统会把该邮箱写入 suppression list，后续发送会自动跳过。

## 7. LLM Settings 大模型配置

该页面用于配置 AI 破冰句和回信分类所使用的 LLM。

### 7.1 Current LLM Configuration

展示当前已配置的 provider：

- Provider
- Status
- Model
- Base URL
- Key 是否已保存，界面只显示脱敏结果
- Updated 更新时间

### 7.2 Protocol 选择

支持两种协议：

OpenAI / Compatible：

- 适用于 OpenAI
- 适用于 DeepSeek
- 适用于其他 OpenAI-compatible endpoint
- 使用 Chat Completions 协议

Anthropic Claude：

- 使用 Anthropic 官方 Messages API
- `system` 是顶层字段
- `messages` 放用户内容
- 必须提供 `max_tokens`

### 7.3 配置字段

- API Key：模型服务商 API key
- Base URL：接口地址
- Model：模型名称
- System Prompt：系统提示词

OpenAI 默认示例：

```text
Base URL: https://api.openai.com/v1
Model: gpt-4o-mini
```

Anthropic 默认示例：

```text
Base URL: https://api.anthropic.com
Model: claude-3-5-haiku-latest
```

DeepSeek 示例：

```text
Base URL: https://api.deepseek.com
Model: deepseek-chat
```

### 7.4 保存规则与安全措施

操作步骤：

1. 选择 OpenAI / Compatible 或 Anthropic Claude。
2. 输入 API Key、Base URL、Model。
3. 按需修改 System Prompt。
4. 点击 Save LLM Settings。

安全说明：

- API Key 使用密码输入框。
- API Key 不会在表格里明文显示。
- 保存后只显示脱敏预览。
- 安装 `cryptography` 后，会加密存储到本地数据库。
- 如果 API Key 留空，会保留已有 key。

## 8. 推荐使用流程

首次使用建议按以下顺序：

1. 进入 LLM Settings，配置 OpenAI-compatible 或 Anthropic Claude。
2. 进入 Dispatch Control，添加 Mailforge 发件箱。
3. 进入 Security Monitor，添加 seed 测试邮箱。
4. 回到 Dispatch Control，先执行 Managed Gmail Placement Test。
5. 用 1 到 3 条测试 lead 发送小批量邮件。
6. 进入 Audit Logs 检查邮件最终 HTML。
7. 进入 Shared Inbox 同步回信。
8. 进入 Security Monitor 查看退信、退订、seed spam 指标。
9. 确认一切正常后，再扩大名单发送。

## 9. 常见问题

### 9.1 为什么发不出去？

请检查：

- 发件箱是否已保存
- 发件箱状态是否为 active
- 密码是否为正确的 App Password
- SMTP host / port 是否正确
- 是否达到 daily limit
- 目标邮箱是否在 suppression list
- 目标域名是否达到每日限制

### 9.2 为什么 AI 破冰句没有生效？

请检查：

- LLM Settings 是否保存了 API key
- 当前 provider 是否为 active
- Base URL 是否正确
- Model 名称是否正确
- 发送页面是否勾选 AI realtime icebreaker

### 9.3 为什么 Gmail 落箱测试没有结果？

请检查：

- 是否完成 ePetrel 授权
- BFF 地址是否配置正确
- 测试邮件是否真实发送成功
- BFF 是否收到 Gmail 检测服务回写
- 是否点击 Refresh Placement Result

### 9.4 为什么收件箱没有回信？

请检查：

- 发件箱是否配置了 IMAP host / port
- 发件箱密码是否支持 IMAP 登录
- 回信是否进入了 INBOX
- Recent emails per mailbox 是否设置太小

## 10. 旧文件清理建议

新版控制台运行时必须保留：

- `web_app.py`
- `templates/`
- `static/`
- `config.py`
- `requirements.txt`
- `modules/`
- `database/`

可以删除但建议先备份的旧文件：

- `app.py`：旧版 Streamlit UI 入口。新版 FastAPI 控制台不依赖它。确认不再需要回滚 Streamlit 后可删除。
- `Doc/p1.html`
- `Doc/p2.html`
- `Doc/p3.html`
- `Doc/p4.html`

这些 `p1-p4.html` 是 UI 参考稿，当前新版模板已经吸收其布局风格，运行时不再依赖。

可选保留的辅助文件：

- `check_email/send_test.py`：SMTP 独立测试脚本，排查发信问题时有用。
- `check_email/check_warm_function.py`：IMAP 独立测试脚本，排查收信或 warmup 时有用。
- `Doc/新需求.md`：需求记录，不参与运行，可作为项目历史保留。

如果删除 `app.py`，还可以从 `requirements.txt` 中移除：

```text
streamlit>=1.30.0
```

注意：不要删除本地数据库和密钥文件，除非你确认要清空所有配置和历史数据。

常见需要保留的数据文件包括：

- `database/storage.db`
- `.env`
- `.epetrel_secret.key`
