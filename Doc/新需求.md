### 1. 发件箱导入与模板下载

- 手工新增发件箱表单也已补齐并强制填写 SMTP/IMAP Host 与 Port，登录检测会使用用户填写的配置。
- 页面新增“Download sender template / 下载发件箱模板”链接：`/senders/template`。
- 你制作的模板文件请放到：`static/templates/senderemaillist.xlsx`。如果该文件不存在，系统会临时生成一个带 Gmail/Outlook 示例行的模板供下载。
- 页面已加入 Gmail / Workspace 与 Outlook / Microsoft 365 的 SMTP/IMAP 参考说明：
  - Gmail SMTP：`smtp.gmail.com`，465 SSL 或 587 STARTTLS/TLS。
  - Gmail IMAP：`imap.gmail.com`，993 SSL/TLS。
  - Outlook/Microsoft 365 SMTP：`smtp.office365.com` 或 `smtp-mail.outlook.com`，587 STARTTLS。
  - Outlook/Microsoft 365 IMAP：`outlook.office365.com` 或 `imap-mail.outlook.com`，993 SSL/TLS。

### 2. Managed Gmail Placement Test
- 客户端已从“测试全部 active 发件箱”改为“每个发件域名只选一个 active 发件箱测试一次”。
- 客户端新增本地域名每日计数表 `email_test_domain_counters`，同一发件域名每天最多测试 3 次。
- BFF 在创建 placement request 时会同步调用 WP：`POST /wp-json/aiinked/v1/email-test/requests`，由 WP 端再次按域名每日 3 次进行数据库限制。

### 3. WP Gmail API 落库检测
- WP 新增文件：`includes/aiinked_email_test.php`。
- WP 新增数据库表：
  - `wp_aiinked_email_test_requests`：记录 emailtestrequestid、用户、发件邮箱、域名、Gmail message id、placement、状态等。
  - `wp_aiinked_email_test_domain_limits`：记录同一域名每天测试次数。
- WP 新增内部接口：
  - `POST /wp-json/aiinked/v1/email-test/requests`：BFF 创建请求时写入 WP 并执行域名每日 3 次限制。
  - `POST /wp-json/aiinked/v1/email-test/scan`：内部触发 Gmail API 扫描。
- WP 新增每分钟 cron：扫描托管 Gmail 最近邮件，根据 `X-ePetrel-EmailTestRequestId`、subject、snippet 匹配 request id，读取 Gmail `labelIds`：
  - 含 `SPAM` => `spam`
  - 含 `INBOX` 且不含 `SPAM` => `inbox`
  - 其他 => `unknown`
- 检测完成后 WP 回写 BFF：`POST /v1/email-test/results`。
- WP 需要配置：
  - `AIINKED_EMAIL_TEST_GMAIL_ADDRESS`
  - `AIINKED_EMAIL_TEST_GMAIL_CLIENT_ID`
  - `AIINKED_EMAIL_TEST_GMAIL_CLIENT_SECRET`
  - `AIINKED_EMAIL_TEST_GMAIL_REFRESH_TOKEN`
  - `AIINKED_INTERNAL_TOKEN`
  - `AIINKED_FRAMER_BFF_BASE_URL`

### 4. Configure Copy Variants
- 已读取 `Doc/变体研究.md` 并落实到 `Configure Copy Variants`。
- 情况 A：用户输入带 `{Name}`、`{Company}`、`{Company_Bio}`、`{Position}` 等变量的纯文本后，点击 “AI Generate Variants / AI 自动生成变体”，客户端同步调用当前 LLM 设置生成 `{A|B|C}` Spintax，并用完整生成内容替换输入框原文。
- 情况 B：用户自己准备 `{A|B|C}` 纯文本变体时，不调用 LLM；系统会检测大括号与 Spintax 空选项格式，通过后按变体发送。
- 发送时按 `sender_email + sequence_no` 作为 Spintax seed 渲染，降低同一发件箱连续邮件内容重复概率。
- LLM Settings 的默认系统提示词已改为面向冷邮件变体生成，并在提示词下方增加用途说明和修改风险提醒。

### 5. 默认 P.S. 信誉保护文案
- 发送时默认追加可变体化 P.S.：
  - `P.S. {If you're not the right person for this, just reply with 'No' and I'll take you off the list.|If this is not your area, reply with 'No' and I'll remove you from the list.|Wrong contact? Just reply 'No' and I'll take you off the list.}`
- 如果用户正文中已经包含类似 `not the right person` 或 `reply with 'No'` 的内容，则不会重复追加。
- 输入框下方已加入提示：系统会默认追加这段礼貌拒绝/退订文案以保护邮箱信誉，也允许用户自行写成变体。
